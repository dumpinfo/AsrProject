Function

   

This program will read one or more input gram files and generate/update a back-off -gram language model as described in section 14.5. The -n option specifies the order of the final model. Thus, to generate a trigram language model, the user may simply invoke the tool with -n 3 which will cause it to compute the FoF table and then generate the unigram, bigram and trigram stages of the model. Note that intermediate model/FoF files will not be generated.

As for all tools which process gram files, the input gram files must each be sorted but they need not be sequenced. The counts in each input file can be modified by applying a multiplier factor. Any -gram containing an id which is not in the word map is ignored, thus, the supplied word map will typically contain just those word and class ids required for the language model under construction (see LSUBSET).

LBUILD supports Turing-Good and absolute discounting as described in section 14.3.1.
